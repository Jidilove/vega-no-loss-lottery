# No-Loss Lottery (DeFi протокол)

## Обзор

Данный проект реализует **DeFi-протокол типа no-loss lottery**.

Пользователи вносят токены в общий пул.  
Эти средства инвестируются во внешний vault и приносят доход.  
После завершения периода (epoch) полученная прибыль распределяется случайным образом между участниками.

**Ключевое свойство:**
- пользователи не теряют свой депозит
- разыгрывается только доход (yield)

---

## Основная идея

В отличие от классической лотереи:
- участники не формируют выигрыш напрямую
- выигрыш возникает за счёт внешней доходности

\[
Y = A_T - D
\]

где:
- \(D\) — сумма всех депозитов
- \(A_T\) — итоговый баланс после инвестирования

---

## Вероятностная модель

Вероятность выигрыша пропорциональна депозиту пользователя:

\[
P_i = \frac{d_i}{D}
\]

где:
- \(d_i\) — депозит пользователя
- \(D\) — общий депозит

---

## Архитектура

Система состоит из следующих компонентов:

- **ERC20 токен** — используется для депозитов (MockUSDC)
- **ERC4626 vault** — отвечает за генерацию доходности
- **NoLossLottery** — основной контракт (логика лотереи)
- **Randomness Provider** — источник случайности (mock)
- **ERC721 NFT** — фиксирует результат розыгрыша

## PS

- Лотерея **не vault**
- использует внешний vault для получения дохода

---

## Жизненный цикл - эпоха (Epoch)

Каждый раунд лотереи называется **epoch**.

### Последовательность работы:

1. Администратор запускает epoch (задаёт длительность)
2. Пользователи вносят депозиты
3. Средства отправляются во vault
4. Vault генерирует доход
5. Epoch завершается (по времени)
6. Запрашивается случайное число
7. Определяется победитель
8. Происходит финализация
9. Пользователи выводят депозиты
10. Победитель получает приз и NFT

---

## Выбор победителя

Выбор происходит через накопленное распределение:

```solidity
uint256 target = randomWord % totalPrincipal;

for (uint256 i = 0; i < deposits.length; i++) {
    cumulative += deposits[i];
    if (target < cumulative) {
        return i;
    }
}

## Интерпретация

* каждый пользователь занимает “отрезок” пропорционально своему депозиту
* случайное число выбирает точку на этом отрезке
* пользователь, в чей интервал попало число — выигрывает

⸻

### Финал

После завершения epoch:
uint256 assets = vault.previewRedeem(epoch.totalShares);
uint256 prize = assets - epoch.totalPrincipal;

* определяется общий баланс
* вычисляется доход
* назначается победитель
